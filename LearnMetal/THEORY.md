# Уровень 0 — Фундаментальные основы

## Как работает шейдер

```metal
#include <metal_stdlib>
using namespace metal;

[[ stitchable ]] half4 myShader(float2 position, half4 color, float2 size) {
    // position — координата текущего пикселя (в пикселях)
    // color    — исходный цвет пикселя
    // size     — размер view (передаётся из Swift)
    return half4(r, g, b, 1.0);
}
```

Каждый пиксель обрабатывается **параллельно и независимо**. Шейдер получает координату, возвращает цвет.

### Аргументы шейдера

`position` и `color` передаются **SwiftUI автоматически** при `.colorEffect()`.
Всё что передаёшь из Swift — идёт **после** этих двух:

```
half4 Flag(float2 position, half4 color, float2 size)
             ↑               ↑             ↑
          автоматом       автоматом     ты передал
```

## Пиксельная координатная система

Отсчёт от **левого верхнего угла**:

```
(0, 0) ──────────── (size.x, 0)
  │                       │
  │                       │
(0, size.y) ──── (size.x, size.y)
```

- `position.x` растёт **слева направо**
- `position.y` растёт **сверху вниз**

---

## Нормализация

Превращает любое значение в диапазон **0.0 — 1.0**:

```
normalized = value / maximum
```

| Выражение | Максимум | Результат |
|---|---|---|
| `position.x` | `size.x` | 0.0 (левый край) → 1.0 (правый) |
| `position.y` | `size.y` | 0.0 (верх) → 1.0 (низ) |
| `x + y` | `size.x + size.y` | диагональный градиент |
| `x * y` | `size.x * size.y` | квадратичный градиент |

**Правило**: максимум зависит от операции в числителе. Подставь максимальные значения каждой переменной и посчитай.

**Важно**: нельзя смешивать пиксели и доли в одной формуле:
```
position.x  vs  size.x * 0.3     ← оба в пикселях ✓
uv.x        vs  0.3              ← оба в долях ✓
position.x  vs  uv.x * 0.3      ← пиксели vs доли ✗
```

## UV-координаты

Нормализованные координаты пикселя:

```metal
float2 uv = position / size;  // uv.x: 0→1, uv.y: 0→1
```

Удобны для градиентов — значения от 0 до 1 совпадают с диапазоном цвета.

## Цветовое пространство RGBA

```metal
half4(R, G, B, A)  // каждый канал от 0.0 до 1.0
```

| Цвет | R | G | B |
|---|---|---|---|
| Чёрный | 0 | 0 | 0 |
| Белый | 1 | 1 | 1 |
| Красный | 1 | 0 | 0 |
| Жёлтый | 1 | 1 | 0 |
| Голубой | 0 | 1 | 1 |
| Серый 50% | 0.5 | 0.5 | 0.5 |

A (альфа) = прозрачность. 1.0 = непрозрачный, 0.0 = полностью прозрачный.

Белый — это не отдельный цвет, это **все три канала включены** (R=1, G=1, B=1).

Когда все три канала **равны** — получается оттенок серого. Поэтому `half4(brightness, brightness, brightness, 1.0)` — серый нужной яркости.

### Broadcasting

Metal автоматически "размножает" одно число на все компоненты:
```metal
return mask;  // = half4(mask, mask, mask, mask)
```
**Осторожно**: 4-й компонент — это альфа! Если `mask = 0`, пиксель станет полностью прозрачным, а не просто чёрным.

# Уровень 1 — Формы (SDF)

## Системы координат

В уровне 0 мы делали `position / size`, получали UV и сразу использовали как цвет. Это работало, потому что градиенты и полосы не требуют точного позиционирования. Но как только мы хотим нарисовать фигуру — круг, квадрат, линию — нам нужно отвечать на вопрос: "где она находится?" А для этого надо разобраться, в какой системе координат мы работаем.

### Пиксельная система (position)

Это то, что Metal даёт по умолчанию. Каждый пиксель знает свою позицию на экране в пикселях:

```
(0, 0) — левый верхний угол
(390, 0) — правый верхний
(0, 844) — левый нижний
(390, 844) — правый нижний
```

Ноль — в левом верхнем углу. X растёт вправо, Y — вниз. Значения зависят от размера экрана. На iPhone SE `size.x = 320`, на iPhone 15 Pro — `393`. Один и тот же шейдер будет давать разные результаты на разных устройствах, если работать напрямую с `position`.

### UV-система (position / size)

Делим на размер — получаем доли:

```
(0.0, 0.0) — левый верхний
(1.0, 0.0) — правый верхний
(0.0, 1.0) — левый нижний
(1.0, 1.0) — правый нижний
```

Удобно для градиентов — значения от 0 до 1 совпадают с диапазоном цвета. Но для фигур есть проблема: центр экрана находится в точке `(0.5, 0.5)`. Это не ноль. Каждый раз, когда мы хотим что-то разместить в центре, приходится вычитать 0.5.

И ещё одна проблема. На экране 390×844 один "шаг" по `uv.x` — это 390 пикселей, а по `uv.y` — 844 пикселя. Если мы нарисуем фигуру, которая одинакова по X и Y в UV — на экране она будет растянута по вертикали. Круг станет овалом.

### Центрированная система

Сдвигаем начало координат в центр экрана:

```metal
float2 centered = position - size / 2.0;
```

Каждый пиксель теперь знает **насколько далеко он от центра**:
```
(-195, -422) — левый верхний
( 195, -422) — правый верхний
(-195,  422) — левый нижний
( 195,  422) — правый нижний
(   0,    0) — центр экрана
```

Центр — в нуле. Отрицательные значения — слева и сверху, положительные — справа и снизу. Значения в пикселях — значит пропорции правильные, круг останется кругом.

### Какую систему выбрать?

| Задача | Система | Почему |
|---|---|---|
| Градиент | UV | Значения совпадают с цветом (0–1) |
| Круг, формы | Центрированная пиксельная | Центр в нуле, пропорции верные |
| Точные размеры (50px) | Пиксельная | Прямой контроль размера |

**Стандарт в индустрии:** для фигур используют пиксельную систему. Параметры (радиус, размер) передаются в поинтах — как в Figma, SwiftUI, UIKit. Круг диаметром 100 в Figma = `radius = 50` в шейдере.

### Aspect Ratio (соотношение сторон)

Если экран не квадратный, круг в UV превращается в овал. `uv.x` идёт от 0 до 1 на 390px, а `uv.y` — от 0 до 1 на 844px. Один "шаг" по X и по Y — разное количество пикселей.

**Решение 1** — работать в пикселях (рекомендуется):
```metal
float2 center = size / 2.0;
float dist = length(position - center);  // пропорции верные автоматически // dist — это distanation ?
```

**Решение 2** — скорректировать UV:
```metal
float aspect = size.x / size.y;
float2 centered = float2((uv.x - 0.5) * aspect, uv.y - 0.5);
```

В пикселях aspect ratio не нужен — пропорции экрана уже учтены.

## SDF — Signed Distance Function

### Что мы уже знаем

В задании 0.4 мы считали расстояние от пикселя до центра экрана:

```metal
float dist = length(uv - float2(0.5, 0.5));
```

И использовали его как яркость — чем дальше от центра, тем темнее. Получился круговой градиент.

По сути мы уже тогда построили **поле расстояний** — для каждого пикселя экрана мы знали, насколько далеко он от центра. Но мы использовали это поле только одним способом — как градиент. SDF — это тот же подход, но обобщённый на любые фигуры и с гораздо большим набором применений.

### От расстояния к фигуре

`length(p - center)` даёт расстояние от пикселя до точки. Но точка — это не фигура. Как из расстояния до точки получить круг?

Круг — это множество точек, которые находятся на расстоянии `r` от центра. Не дальше, не ближе — ровно `r`. Если мы из расстояния вычтем радиус:

```metal
float sdf = length(centered) - radius;
```

...то получим новое значение, у которого появился смысл:

```
Пиксель ровно на окружности:
    length = radius    →  sdf = radius - radius = 0

Пиксель внутри круга:
    length < radius    →  sdf = что-то отрицательное

Пиксель снаружи:
    length > radius    →  sdf = что-то положительное
```

Ноль — это граница. Минус — внутри. Плюс — снаружи. Вот что значит "signed" — **со знаком**. Знак говорит, по какую сторону от границы мы находимся.

### Поле расстояний — как топографическая карта

Представь топографическую карту, где вместо высоты — расстояние до края фигуры. Граница фигуры — это "уровень моря" (ноль). Внутри — "под водой" (минус). Снаружи — "над водой" (плюс).

Если бы мы раскрасили `sdf` как высоту:

```
Далеко снаружи (+100):  ярко-белый
Близко снаружи (+5):    чуть светлее серого
Граница (0):            серый
Близко внутри (-5):     чуть темнее серого
Далеко внутри (-100):   чёрный
```

### Визуализация SDF (отладочный приём)

Можно буквально отобразить SDF как градиент, чтобы увидеть поле расстояний глазами:

```metal
float viz = sdf / (radius * 2.0) + 0.5;
return half4(viz, viz, viz, 1.0);
```

Делим на `radius * 2.0` — сжимаем диапазон до маленьких чисел. Прибавляем `0.5` — чтобы граница (sdf = 0) была серой. Делитель `radius * 2.0` подобран так, чтобы центр круга (sdf = -radius) давал 0.0 (чёрный), а точка на расстоянии radius от края давала 1.0 (белый).

Это **не** круг — это плавный градиент по всему экрану, показывающий все значения SDF. Полезно для отладки — всегда можно "посмотреть" на SDF глазами и понять, правильно ли посчитана геометрия.

### Числовой пример

Круг радиусом 100px в центре экрана (195, 422):

**Пиксель в самом центре** `(195, 422)`:
```
centered = (195, 422) - (195, 422) = (0, 0)
length(0, 0) = 0
sdf = 0 - 100 = -100     ← глубоко внутри, в 100px от края
```

**Пиксель на полпути к краю** `(245, 422)`:
```
centered = (245, 422) - (195, 422) = (50, 0)
length(50, 0) = 50
sdf = 50 - 100 = -50     ← внутри, в 50px от края
```

**Пиксель на границе** `(295, 422)`:
```
centered = (295, 422) - (195, 422) = (100, 0)
length(100, 0) = 100
sdf = 100 - 100 = 0      ← ровно на окружности
```

**Пиксель чуть за границей** `(298, 422)`:
```
centered = (298, 422) - (195, 422) = (103, 0)
length(103, 0) = 103
sdf = 103 - 100 = 3      ← снаружи, в 3px от края
```

**Пиксель в углу экрана** `(0, 0)`:
```
centered = (0, 0) - (195, 422) = (-195, -422)
length(-195, -422) = 465
sdf = 465 - 100 = 365    ← далеко снаружи
```

Значение `sdf` — это всегда реальное расстояние до ближайшей точки на границе, в пикселях. Пиксель с `sdf = 3` находится ровно в 3 пикселях от окружности. Пиксель с `sdf = -50` — в 50 пикселях от окружности, но с внутренней стороны.

### Почему не просто if/else?

Можно было бы написать:
```metal
if (length(centered) < radius) return white;
else return black;
```

И это бы работало. Но ты получишь только два состояния — внутри или снаружи. Жёсткий край с пиксельной лесенкой. Никаких плавных переходов, свечений, обводок.

SDF хранит **насколько далеко** пиксель от края. Имея расстояние, можно решать, что с ним делать — один `sdf`, множество разных визуальных эффектов:

| Эффект | Код |
|---|---|
| Резкий край | `step(0.0, sdf)` |
| Плавный край | `smoothstep(-1.0, 1.0, sdf)` | как работает smoothstep?
| Обводка | `smoothstep(-1.0, 1.0, abs(sdf) - thickness)` |
| Свечение | `1.0 / (1.0 + sdf * sdf * 0.01)` |

### SDF для других фигур

Круг — простейший случай. Но принцип одинаковый для любой фигуры: функция принимает координату пикселя и возвращает расстояние до ближайшего края со знаком. Прямоугольник, линия, треугольник, звезда — для каждой фигуры есть своя формула SDF.

### Комбинирование SDF

Самая мощная возможность — комбинирование фигур простой арифметикой:

```metal
float sdf1 = circleSDF(p);
float sdf2 = rectSDF(p);

float unionSDF     = min(sdf1, sdf2);      // объединение (ИЛИ)
float intersectSDF = max(sdf1, sdf2);      // пересечение (И)
float subtractSDF  = max(sdf1, -sdf2);     // вычитание
```

Почему `min` — это объединение? Если пиксель внутри хотя бы одной фигуры, то хотя бы один `sdf` отрицательный, и `min` вернёт отрицательное число. Пиксель будет считаться "внутри".

## smoothstep — подробный разбор

### Проблема

У нас есть `sdf` — число, которое может быть от -200 до +200. Нам нужно превратить его в цвет — число от 0 до 1.

### Сравнение со step

**step (резкий):**
```metal
float mask = step(0.0, sdf);
// sdf < 0  →  0
// sdf >= 0 →  1
```
Переход мгновенный — один пиксель белый, соседний чёрный. Край с лесенкой (aliasing).

**smoothstep (плавный):**
```metal
float mask = smoothstep(edge0, edge1, sdf);
```
То же что step, но с зоной перехода. Между `edge0` и `edge1` значение плавно меняется от 0 к 1.

### Как работает

```
smoothstep(-2.0, 2.0, sdf)

sdf:       ... -5  -3  -2   -1    0    1    2   3   5 ...
                        ↓                   ↓
результат:  0   0   0  0.15  0.5  0.85  1   1   1
                        ↑                   ↑
                      edge0               edge1
```

Всё что левее edge0 — гарантированный 0.
Всё что правее edge1 — гарантированный 1.
Между ними — плавная S-образная кривая (не линейная).

### Что значат edge0 и edge1 для SDF

`sdf = 0` — граница фигуры. Значит:

- `edge0 = -2.0` означает: "начни переход за 2 пикселя до границы (внутри фигуры)"
- `edge1 = 2.0` означает: "закончи переход через 2 пикселя после границы (снаружи)"

Ширина перехода = `edge1 - edge0` — это **зона размытия** края.

```metal
smoothstep(-1.0, 1.0, sdf)    // 2px размытие — чёткий, но без лесенки
smoothstep(-5.0, 5.0, sdf)    // 10px размытие — заметно мягкий край
smoothstep(-0.5, 0.5, sdf)    // 1px — минимальное сглаживание
smoothstep(0.0, 0.001, sdf)   // почти резкий (как step)
```

Единицы edge0 и edge1 — те же, что у sdf. Если sdf в пикселях — то и edge в пикселях.

## Собираем круг — итоговый пример

```metal
[[ stitchable ]]
half4 Circle(float2 position, half4 color, float2 size, float diameter) {
    // 1. Центрируем координаты
    float radius = diameter / 2.0;
    float2 centered = position - size / 2.0;

    // 2. Считаем SDF круга
    float sdf = length(centered) - radius;

    // 3. Превращаем SDF в маску (0 внутри, 1 снаружи)
    float mask = smoothstep(-1.0, 1.0, sdf);

    // 4. Выбираем цвет по маске
    half4 white = half4(1, 1, 1, 1);
    half4 black = half4(0, 0, 0, 1);
    return mix(white, black, mask);
}
```

Каждая строка — одна глава из теории. Без магии.
